# 📚 마이그레이션 가이드

## 기존 CMS에서 WordPress로 전환

이 가이드는 CloudPress 기존 사용자가 새로운 100% WordPress 기반 시스템으로 마이그레이션하는 방법을 설명합니다.

---

## 📊 마이그레이션 전략

### 타입별 마이그레이션

#### 1. 전체 백업 복원 (권장)
```
기존 호스팅 (모든 파일, DB, 설정)
         ↓
    .zip 포장
         ↓
   WordPress 업로드
         ↓
   자동 추출 및 설정
```

**장점:** 가장 간편함, 모든 데이터 보존
**시간:** 5분 ~ 30분 (크기에 따라)

#### 2. 선택적 마이그레이션
```
기존 호스팅
    ├─ DB 추출 (.sql)
    ├─ wp-content 추출 (.zip)
    ├─ wp-plugins 추출
    └─ wp-themes 추출
         ↓
    각각 업로드
```

**장점:** 세부 제어 가능
**시간:** 10분 ~ 1시간

#### 3. 콘텐츠만 마이그레이션
```
기존 호스팅의 게시물/페이지
         ↓
  WordPress 임포트 플러그인 XML
         ↓
   새 WordPress에 업로드
```

**장점:** 깔끔한 환경, 선택적 콘텐츠
**시간:** 15분 ~ 2시간

## 🚀 단계별 마이그레이션

### Step 1: 기존 호스팅 백업

#### Option A: 관리자 패널에서
```
1. 기존 호스팅 대시보드 로그인
2. Tools → Export
3. 백업 형식 선택:
   - All content (.xml)
   - Database dump (.sql)
   - File backup (.zip)
4. 다운로드
```

#### Option B: FTP/SFTP로
```bash
# 모든 WordPress 파일 다운로드
sftp user@host.com
cd public_html
mget -R *
quit

# 데이터베이스 덤프 (MySQL)
mysqldump -u username -p database_name > backup.sql
```

#### Option C: 자동 백업 플러그인
```
1. UpdraftPlus 또는 BackWPup 설치
2. Full backup 실행
3. 다운로드 링크 복사
```

### Step 2: 새 WordPress 인스턴스 생성

```
1. CloudPress 대시보드 로그인
2. 호스팅 관리 → 호스팅 생성
3. 도메인 선택
4. WordPress 버전 선택 (최신 권장)
5. 생성 완료
```

**생성된 리소스:**
- ✅ WordPress 인스턴스 (PHP-WASM)
- ✅ SQLite 데이터베이스
- ✅ 관리자 계정 (자동 생성)
- ✅ Cloudflare 통합 (자동)

### Step 3: 마이그레이션 파일 업로드

호스팅 상세 페이지에서:

```
1. WordPress 상세 → 마이그레이션 & 복원
2. 파일 드래그 앤 드롭 또는 선택
3. 지원 형식:
   - .zip (백업 전체)
   - .tar.gz (압축 백업)
   - .sql (데이터베이스만)
   - .json (마이그레이션 데이터)
4. 업로드 시작
```

### Step 4: 마이그레이션 처리

시스템이 자동으로:

```
1. 파일 검증 (형식, 크기, 완전성)
2. 압축 해제 (필요시)
3. WordPress 구조 감지
4. 플러그인/테마 분석
5. 데이터베이스 임포트
6. 사용자 계정 매핑
7. 설정 마이그레이션
8. 메디어 재등록
9. 퍼머링크 재구성
10. 캐시 초기화
```

**진행 상황:** 실시간 로그에서 확인

### Step 5: 마이그레이션 검증

```
1. WordPress 관리자 로그인
   https://example.com/wp-admin/
   
2. 콘텐츠 확인
   - 게시물 수 확인
   - 페이지 확인
   - 미디어 라이브러리
   
3. 설정 확인
   - 사이트 URL
   - 홈 URL
   - 퍼머링크 구조
   - 시간대 및 언어
   
4. 플러그인 확인
   - 활성 플러그인
   - 비호환 플러그인 (경고)
   
5. 테마 확인
   - 활성 테마
   - 테마 커스터마이징
```

### Step 6: 도메인 DNS 업데이트 (필요시)

기존 도메인 사용:

```
현재 DNS 설정
    ↓
Cloudflare로 변경
    ↓
Workers 라우트 설정
    ↓
즉시 적용 (5분~30분)
```

**DNS 레코드:**
```
Type: CNAME
Name: example.com
Value: example.com.cdn.cloudflare.net
```

## ⚠️ 주의사항

### 호환성 문제

#### 호환되는 플러그인
```
✅ WP-Rocket (캐싱)
✅ Yoast SEO
✅ WooCommerce
✅ Contact Form 7
✅ Jetpack
✅ WordPress SEO
✅ MonsterInsights
```

#### 호환되지 않는 플러그인
```
❌ 서버 리소스 집약적 플러그인
   - 무거운 백업 도구
   - CPU/메모리 많이 쓰는 플러그인
   
❌ 독점 호스팅 플러그인
   - cPanel 통합
   - WHM 관련
   
❌ 파일 시스템 직접 접근
   - 커스텀 파일 매니저
   - 고급 FTP 도구
```

### 데이터 손실 방지

1. **백업 검증**
   ```bash
   # ZIP 파일 확인
   unzip -t backup.zip
   
   # SQL 파일 확인
   head -20 backup.sql
   ```

2. **테스트 마이그레이션**
   - 별도 테스트 WordPress에서 먼저 시도
   - 오류 확인 후 프로덕션 진행

3. **단계별 롤백**
   - 각 단계 완료 후 스냅샷 저장
   - 문제 발생 시 이전 버전으로 복원

## 🔧 문제 해결

### 마이그레이션 실패

| 오류 | 원인 | 해결 |
|------|------|------|
| "파일이 손상됨" | ZIP 파일 깨짐 | 다시 다운로드, WinRAR로 검증 |
| "데이터베이스 오류" | SQL 호환성 | 다른 WordPress 버전 시도 |
| "플러그인 오류" | 호환성 없음 | 플러그인 비활성화 후 재시도 |
| "파일 업로드 실패" | 크기 초과 | 파일 압축하거나 선택적 마이그레이션 |
| "타임아웃" | 처리 시간 초과 | 더 작은 백업으로 시도 |

### 성능 문제

```
마이그레이션 후 느린 경우:

1. 캐싱 재설정
   WordPress 상세 → 캐싱 설정 → 저장
   
2. 플러그인 최적화
   비필요 플러그인 비활성화
   
3. 데이터베이스 최적화
   Database Cleaner 플러그인 사용
   
4. 이미지 최적화
   Smush 또는 TinyPNG 플러그인
   
5. Cloudflare 규칙 재설정
   호스팅 상세 → CDN 관리
```

### URL/퍼머링크 오류

```bash
# WordPress 관리자에서:
Settings → Permalinks → 저장

# 또는 .htaccess 재생성:
Permalink 설정 변경 후 저장
```

## 💡 팁

### 마이그레이션 시간 단축

1. **선택적 마이그레이션**
   - 최신 게시물만 마이그레이션
   - 오래된 콘텐츠는 별도 아카이브

3. **CDN 캐시 사전 활성화**
   - 마이그레이션 전 CDN 설정
   - 첫 요청 응답 시간 단축

### 다운타임 최소화

```
현재 상황:
기존 사이트 (live)
    ↓
신규 WordPress (준비)
    ↓
마이그레이션 (30분)
    ↓
DNS 변경 (5~30분)
    ↓
새 사이트 (live)

총 다운타임: 5분 ~ 1시간
```
